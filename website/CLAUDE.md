## Codemap

TanStack Start app with edge SSR on Cloudflare Workers. Deadlock API's production frontend.

```
tanstack-start
├── src
│   ├── router.tsx              # createRouter(): preload=intent, scrollRestoration on
│   ├── routeTree.gen.ts        # auto-generated by tanstack-router-vite-plugin
│   ├── routes/__root.tsx       # <html lang="en">, links tailwind.css, devtools in dev
│   ├── routes/                 # file-based routes
│   ├── styles/tailwind.css     # Tailwind v4 theme
│   ├── components/ui/          # shadcn primitives
│   └── lib/utils.ts            # cn() helper
├── content/blog/               # markdown blog posts (bundled at build time via Vite glob)
├── public
│   └── _headers                # Cloudflare security headers
├── vite.config.ts              # @tanstack/react-start plugin + Tailwind v4 + React Compiler (prod only)
├── wrangler.jsonc              # Workers config: main=dist/server/server.js, assets=dist/client
├── components.json             # shadcn aliases (~/ -> src/)
├── .oxlintrc.jsonc / .oxfmtrc.json
└── tsconfig.json
```

## Build pipeline

`vite build` produces two outputs:

- `dist/client/` — static client bundle + copied `public/` files (`_headers`, `favicon.ico`, etc.)
- `dist/server/server.js` — Workers-compatible Module Worker (`export default { fetch }`).

The server entry is sourced from `@tanstack/react-start/dist/plugin/default-entry/server.ts` and wraps `createStartHandler(defaultStreamHandler)`. It uses `node:async_hooks` and `node:stream`, which requires the `nodejs_compat` flag in Workers.

## Cloudflare Workers

`wrangler.jsonc` declares:

- `main: ./dist/server/server.js` — the SSR handler.
- `assets.directory: ./dist/client` with `binding: ASSETS` — Cloudflare's Workers Static Assets serves `/assets/*`, `/favicon.ico`, etc. directly from edge; unmatched paths fall through to the Worker (HTML routes).
- `compatibility_flags: ["nodejs_compat"]` — required by the SSR handler's Node API usage.
- `_headers` in `public/` is copied into `dist/client/_headers` by Vite and parsed by Cloudflare's Static Assets runtime (security headers, CSP, streamkit X-Frame-Options).

### Local dev

- `pnpm dev` — Vite dev server on :3000 (fast HMR, no Workers runtime).
- `pnpm wrangler:dev` — runs `wrangler dev` against the built artifacts (workerd runtime, matches prod). Re-run `pnpm build` first if sources changed.

Verify SSR end-to-end:

```
pnpm build && pnpm wrangler:dev --port 8787
curl -s http://127.0.0.1:8787/ | head    # body contains rendered <h1>, not just <div id="root">
curl -sI http://127.0.0.1:8787/favicon.ico | grep -i strict-transport  # _headers applied
```

### Deploy

```
pnpm deploy   # = pnpm build && wrangler deploy
```

Requires `wrangler login` (or `CLOUDFLARE_API_TOKEN`) and a Cloudflare account configured for Workers. The worker name lives in `wrangler.jsonc`. Production routes / DNS for `deadlock-api.com` are pointed at this Worker via the Cloudflare dashboard or `wrangler.jsonc#routes`.

## Conventions

- React Compiler runs only in production builds (slow in dev) via `vite-plugin-babel`.
- shadcn primitives live in `src/components/ui/`. Add new ones with `pnpm dlx shadcn@latest add <component>` against the existing `components.json`.
- Tailwind v4: `@plugin` directives in `tailwind.css` for typography/iconify/animations (no JS plugin array).
- TypeScript override `^6.0.3` is enforced via `package.json > pnpm.overrides`.
